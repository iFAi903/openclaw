# 人物复刻分析模板

## 1. Profile
- 姓名 / 常用名
- 角色标签
- 主要领域
- 资料覆盖时间段
- 资料充分度
- 一句话定义

## 2. Thinking Model
- 这个人如何定义问题
- 先看哪些变量
- 常见判断框架
- 典型偏好：第一性原理 / 用户价值 / 商业回报 / 系统效率 / 审美 / 风险控制
- 典型盲区或偏执点

## 3. Action Logic
- 决策快慢
- 信息不充分时怎么决定
- 如何排序优先级
- 是否偏好多做实验、先做再说、长期主义、集中下注
- 对失败、分歧、执行细节的态度

## 4. Communication Style
- 总体语气
- 常见句型
- 节奏：短句 / 长句 / 对话感 / 宣言感
- 是否爱用比喻、故事、数字、反问
- 哪些表达明显不是 ta 会说的

## 5. Heuristics
整理成 3-7 条“遇事会怎么判断”的规则。
每条规则格式：
- 规则
- 解释
- 证据来源
- 置信度

## 6. Boundaries
- 适合借鉴的问题类型
- 不适合借鉴的问题类型
- 哪些结论只在特定阶段/语境成立

## 7. Uncertainty Log
- 缺证据的点
- 来源冲突的点
- 需要补材料的点

## 推荐写法

### 一句话定义
> 他不是在追求更多信息，而是在更快识别“哪个变量真正决定结果”。

### 规则示例
- 规则：先抓主变量，再谈优化。
- 解释：他对复杂问题的第一反应通常不是补全细节，而是找最关键的约束条件。
- 证据来源：某次采访、某篇文章、某段演讲。
- 置信度：高。
