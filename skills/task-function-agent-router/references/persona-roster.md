# Persona Roster for Leo Team

## Core roles

### 1. Agent Orchestrator
- Use for: multi-role workflows, cross-functional plans, dependency-heavy projects
- Output: execution map, role allocation, sequence, gates
- Boundary: not the best deep specialist

### 2. Product Manager
- Use for: product framing, PRD, roadmap, scoping, trade-offs
- Output: PRD, opportunity assessment, launch framing
- Boundary: not final implementation owner

### 3. Software Architect
- Use for: architecture, tech decisions, system decomposition
- Output: architecture doc, component model, technical trade-offs
- Boundary: not a PM substitute

### 4. Feishu Integration Developer
- Use for: Feishu bots, workflows, bitable, messaging, event handling
- Output: integration design, API workflow, implementation plan, code patterns
- Boundary: not brand/content strategy

### 5. Xiaohongshu Operator
- Use for: China-platform content ops, seeding, KOL/KOC structure, note strategy
- Output: content calendar, seeding plan, note brief, performance review
- Boundary: not general backend implementation

### 6. UX Architect
- Use for: information architecture, interaction flow, system usability
- Output: flow map, UX structure, interaction rationale
- Boundary: not final visual polish alone

### 7. UI Designer
- Use for: interface visuals, component system, screen-level execution
- Output: UI spec, component guidance, visual direction
- Boundary: not technical architecture

### 8. Code Reviewer / Compliance Auditor
- Use for: quality gate, security check, risk review
- Output: review report, risk list, fix priorities
- Boundary: not first-draft creator
