# AI天团召唤手册

> 目的：提高角色召唤的效率、命中率与交付质量
> 原则：边用边长，不追求一次写全；凡是高命中 prompt / context，及时沉淀
> 更新方式：每次发现有效召唤格式、稳定上下文、优秀约束条件，就增补进来

---

# 一、通用召唤公式

## 标准公式
**让 [角色] + [动作] + [任务对象] + [目标] + [约束] + [交付物]**

这是命中率最高的通用结构。

## 五要素
1. **角色**：CRO / COO / CTO / CFO
2. **任务**：写、拆、做、算、评估、排查、规划
3. **目标**：想达成什么结果
4. **约束**：时间、预算、风格、边界、范围
5. **交付物**：你最后想拿到什么

## 最稳模板
**让 [角色] 处理 [任务]，目标是 [目标]，面向 [对象]，约束是 [时间/预算/风格/范围]，输出给我 [交付物]。**

# 二、各角色召唤模板

## 1. CRO（品牌 / 文案 / 策略表达）

### 适用场景
- slogan
- 品牌表达
- 传播策略
- 文案与内容创意

### 模板
**让 CRO 为 [项目/产品/活动] 输出 [文案/口号/传播方案/品牌表达]，目标是 [效果]，要求 [风格/受众/时限]。**

### 示例
- 让 CRO 为新产品写一句 slogan，目标是高级感和记忆点强，要求面向 25-35 岁创业者。
- 让 CRO 出一版品牌人格化传播方案，目标是提高用户认同感，要求长期主义，不要短期流量味。

### 高命中补充 context
- 品牌当前阶段（冷启动 / 增长期 / 重塑期）
- 目标受众是谁
- 不要什么风格
- 是否偏短期转化 or 长期品牌资产

---

## 2. COO（运营 / 增长 / 社群 / 电商）

### 适用场景
- 运营方案
- 增长拆解
- 社群策略
- 电商节奏
- 执行排期

### 模板
**让 COO 为 [项目/活动/业务] 做 [运营方案/增长拆解/社群策略/执行计划]，目标是 [指标]，约束是 [预算/周期/资源]。**

### 示例
- 让 COO 做一个社群增长方案，目标是 30 天新增 500 个精准用户，预算控制在 3000 内。
- 让 COO 拆一下这个活动的执行节奏和 ROI，目标是提高转化率，要求分周推进。

### 高命中补充 context
- 当前阶段：拉新 / 激活 / 留存 / 转化
- 核心 KPI：新增、GMV、转化率、留存率
- 可用资源：预算、人手、渠道
- 交付时是否要附节奏表 / KPI 表 / 风险预案

---

## 3. CTO（技术 / 产品实现 / 架构 / 排障）

### 适用场景
- 技术方案
- MVP 实现
- 架构评估
- bug 排查
- 部署与重构

### 模板
**让 CTO 为 [项目/功能/系统] 做 [技术方案/代码实现/架构审查/问题排查]，目标是 [结果]，要求 [技术栈/时限/限制]。**

### 示例
- 让 CTO 为这个功能出技术方案，目标是两天内做出 MVP，要求用 Next.js + Supabase。
- 让 CTO 排查这个报错，目标是找到根因并给出修复方案，要求先止血再定位。

### 高命中补充 context
- PRD 或需求背景
- 现有技术栈
- 不允许改动的范围
- 是否先做 MVP
- 交付物要方案、代码、部署指南，还是排障结论

---

## 4. CFO（财务 / ROI / 风险 / 投资评估）

### 适用场景
- 预算分析
- 财务评估
- ROI 测算
- 风险判断
- 投资建议

### 模板
**让 CFO 对 [项目/决策/预算/投资标的] 做 [财务分析/ROI评估/风险评估/预算建议]，目标是 [结果]，要求 [口径/周期/保守程度]。**

### 示例
- 让 CFO 评估这个项目值不值得投，目标是看回报与风险，要求偏保守。
- 让 CFO 做一版预算分析，目标是控制现金流风险，要求给出乐观/中性/保守三种情景。

### 高命中补充 context
- 决策周期：短期 / 中期 / 长期
- 预算范围
- 风险偏好：保守 / 中性 / 进取
- 是否要情景分析
- 输出是结论型还是表格型

---

# 三、最短可用口令

适合快速发号施令：

- **让 CRO 写**
- **让 COO 拆**
- **让 CTO 做**
- **让 CFO 算**

示例：
- 让 CRO 写这个活动的传播主文案
- 让 COO 拆这个增长方案
- 让 CTO 做这个 MVP
- 让 CFO 算这个项目的 ROI

# 四、提高命中率的 Prompt 补强法

## 补强法 1：加目标
不要只说“做一个方案”，要说“目标是什么”。

差：
- 让 COO 做个运营方案

好：
- 让 COO 做个运营方案，目标是 30 天提升转化率 20%。

## 补强法 2：加边界
边界越清，返工越少。

可加的边界：
- 时间
- 预算
- 风格
- 不要做什么
- 不允许改什么

## 补强法 3：加交付物
明确最后你想拿到什么。

例如：
- 输出给我一页方案
- 输出给我分步骤执行表
- 输出给我代码和部署说明
- 输出给我 ROI 结论和风险点

## 补强法 4：给角色上下文，而不是只给命令
好的召唤，不只是命令，还要补环境。

例如：
- 当前项目阶段
- 面向谁
- 为什么做
- 之前已经做过什么
- 哪些约束不能碰

# 五、推荐上下文模板

## 任务上下文模板
- 背景：
- 目标：
- 面向对象：
- 当前阶段：
- 约束条件：
- 不要做的事：
- 预期交付物：

## 复盘上下文模板
- 这次任务是否命中？
- 哪句话最有效？
- 哪些上下文一补上，质量明显提升？
- 哪些约束加进去后，返工显著减少？
- 是否值得沉淀为固定模板？

# 七、维护原则

1. **只收录实战有效的模板**，不堆空话
2. **每次发现高命中表达，立刻补进来**
3. **优先沉淀“减少返工”的上下文**
4. **优先沉淀“交付物明确”的召唤句式**
5. **让手册越用越短，越短越准**

---

# 一句话总结

**角色 + 任务 + 目标 + 约束 + 交付物**

这是 AI 天团召唤命中率最高的核心骨架。
