# CN Web Search 使用规则

最后更新：2026-03-28
状态：active

## 定位

`CN Web Search` 是一个**中文搜索场景优先**的外部检索 skill。
它适合拿来找中文网页线索、公众号线索、中文财经/技术/百科入口。
它不是默认搜索总入口，也不是高置信结论来源。

## 什么时候该用

优先使用 `CN Web Search` 的场景：

- 用户明确要**搜中文网页**
- 用户要找**中文世界**的信息入口
- 用户提到：公众号、微信文章、知乎、百度、头条、A股、财新、东方财富、集思录、中文百科
- 需要补充**中文舆情 / 中文媒体 / 中文社区**线索
- 普通 `web_search` 结果对中文语境不够好，或需要多中文引擎交叉看线索

一句话判断：
> 当问题的答案更可能埋在中文互联网里，而不是英文互联网里，就优先考虑它。

## 什么时候别用

以下情况**不要优先用** `CN Web Search`：

- 用户只是做**普通通用搜索**，没有中文语境要求
- 需要**英文资料、国际新闻、海外技术文档、英文论文**
- 需要**高可信原始来源**（官网、论文原文、监管披露、正式文档）
- 需要**结构化事实核验**，而不是搜线索
- 需要避免 SEO 污染、广告页、转载页的干扰

一句话判断：
> 如果我要的是“准”，不是“广”；要的是“原始来源”，不是“搜索入口”，那就别先用它。

## 默认路由原则

1. **中文搜索请求** → 优先考虑 `CN Web Search`
2. **英文/国际搜索请求** → 优先用 `web_search`
3. **已知具体网址/文章/官网** → 直接 `web_fetch`
4. **高风险事实（投资/法律/医疗/政策）** → 搜索只做线索，最终必须回到原始来源验证

## 极简操作规则

- 找中文线索：用它
- 找英文资料：别用它
- 找官网原文：直接 `web_fetch`
- 做正式结论：别只靠它，至少二次验证

## 风险提醒

`CN Web Search` 的风险不在本地执行，而在**外部内容质量参差不齐**。
所以它最适合做：
- 发现线索
- 扩展搜索面
- 找中文入口

不适合做：
- 单一来源定论
- 直接采信搜索页摘要
- 在高风险主题上跳过交叉验证
