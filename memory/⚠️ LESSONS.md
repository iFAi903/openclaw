## 📋 核心规则

### 1. 回复策略错误（已踩坑）

#### ❌ 错误：重复回复同一问题

**错误示例**：

- 第1次："快捷指令调用失败，因为..."
- 第2次："已创建ICS文件，请双击导入..."

**正确做法**：

```
快捷指令暂不支持命令行传参，已为你生成ICS文件：
📁 ~/Desktop/文件名.ics
⏰ 时间：XX:XX-XX:XX

导入方法：双击文件 → 确认导入
```

#### ❌ 错误：过度解释

**错误示例**：

- "首先我尝试调用快捷指令..."
- "然后我发现需要参数..."
- "最后我决定生成ICS文件..."

**正确做法**：

- 直接给出结果和解决方案
- 技术细节只在用户询问时说明

#### ❌ 错误：多次确认

**错误示例**：

- "收到你的请求"
- "正在处理..."
- "处理完成，结果是..."

**正确做法**：

- 直接给出最终结果

**✅ 一次回复原则**：
每个用户请求，只回复一次，包含：

1. **结果**：是否成功/失败
2. **关键信息**：文件位置、时间、链接等
3. **下一步**：如果需要用户操作，简要说明

### 2. 任务执行错误（已踩坑）

#### ❌ 错误：AI新闻数量不足

**问题**：只发了10-11条新闻，不符合15条要求
**后果**：不符合任务规范
**解决**：严格计数，必须15条

#### ❌ 错误：热门应用混在新闻里

**问题**：把TOP 5应用混在6大板块新闻中
**后果**：格式错误，结构混乱
**解决**：两个任务完全分开，独立板块

#### ❌ 错误：板块缺失

**问题**：没有按6大板块分类新闻
**后果**：信息组织混乱
**解决**：严格按照6板块划分

#### ❌ 错误：来源不全

**问题**：TOP 5只覆盖部分平台
**后果**：不符合要求
**解决**：必须覆盖全部5个平台

**✅ 执行检查清单**（发送前必查）：

- [ ] 新闻数量 = 15条
- [ ] 板块数量 = 6个
- [ ] TOP 5平台 = 5个（Product Hunt/GitHub/Toolify/Hacker News/Reddit）
- [ ] 每条新闻都有标题、摘要、来源、链接
- [ ] 每个应用都有平台、名称、功能、亮点、链接
- [ ] 底部有一句话总结

## 📅 2026-03-08 安全红线新增

### 5. 发现新 Skill 急于安装（严重错误）

**场景**：
- 用户要求实现功能
- 我发现需要新的 skill
- **立即寻找并准备安装** ← 错误

**后果**：
- Skill 生态混乱，可能包含木马
- 未经审计的 skill 可能导致：
  - 系统被攻破
  - 敏感信息泄露
  - 工作成果瞬间丢失

**解决**：
1. **立即停止** — 不安装、不启用
2. **安全审计** — 提交到 https://ai.gendigital.com/skill-scanner
3. **等待结果** — 确认无风险
4. **报告用户** — 审计结果 + 安装建议
5. **获得授权** — 明确许可后才安装

**核心原则**：
> 效率永远让位于安全，否则一切积累可能瞬间归零。
> 宁可慢，不可错。

### 2. 网站构建语法错误（2026-02-26）

**错误**: `news.ts` 中包含中文引号「」导致构建失败
**错误信息**:

```
Syntax Error: Unexpected character '「'
```

**解决**: 替换为单引号 ''
**预防**:

- 写入文件前检查特殊字符
- 构建失败时优先检查引号问题
- 使用英文标点符号
