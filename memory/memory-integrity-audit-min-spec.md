# 记忆完整性审计脚本最小版说明

## 目标
用最小成本把“正式角色是否具备可用长期记忆”从人工感知，变成可重复检查。

## 本轮最小交付
1. `scripts/memory_integrity_audit.py`
2. `reports/memory-integrity-latest.md`
3. 本说明文件（定义审计范围、通过条件、后续扩展位）

## 审计范围（最小版）
仅检查 `agents/*` 下已存在 `MEMORY.md` 的正式角色目录。

当前目标角色：
- `agents/product-agent`
- `agents/cfo`
- `agents/cro`
- `agents/coo`
- `agents/cto`

## 检查项
### P0 必过
- 角色目录存在 `MEMORY.md`
- 文件非空
- 包含对根记忆的服从/对齐声明
- 包含 6 个核心模块：
  - 角色定义
  - 职责边界
  - 记忆回流规则
  - 核心原则
  - 决策索引
  - 输出范式

### P1 先记报告，不阻断
- 是否同时存在 `SOUL.md`
- 是否同时存在 `IDENTITY.md`
- 是否同时存在 `TOOLS.md`
- 是否同时存在 `USER.md`

## 通过标准
- 所有目标角色的 P0 项全部通过
- 报告中明确列出缺失项与建议动作

## 当前已知约束
- 最小版先不自动发现“所有新 Agent”
- 最小版先不校验内容质量深浅，只校验结构完整性
- 最小版先不写入 LanceDB，只输出 Markdown 报告

## 下一步扩展
- 自动发现新增正式角色并纳入审计
- 增加主权声明关键词白名单/正则
- 增加空占位内容识别（如“等待记录...”过多）
- 将审计结果接入 nightly review
