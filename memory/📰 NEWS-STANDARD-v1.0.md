## ✅ 验收标准清单

### 1. 新闻内容
| 检查项 | 标准 | 状态 |
|--------|------|------|
| 来源数量 | 至少4个不同来源 | ✅ TheVerge 4条 + TechCrunch 4条 + Wired 2条 + 36氪 2条 |
| 中文内容 | 必须包含中文科技媒体 | ✅ 36氪 提供中文内容 |
| 标题翻译 | 中文为主，保留人名/公司/产品英文 | ✅ |
| 摘要质量 | 流畅，无占位符残留 | ✅ 无 `__KEYWORD_X_X__` |

### 2. 热门产品追踪
| 检查项 | 标准 | 状态 |
|--------|------|------|
| 平台覆盖 | 5个平台各1个 | ✅ Product Hunt / GitHub / Toolify / HackerNews / Reddit |
| 摘要长度 | ≤60字，包含定义+功能+场景 | ✅ |
| 标题格式 | 产品名保留英文，附中文描述 | ✅ |

### 3. 技术稳定性
| 检查项 | 标准 | 状态 |
|--------|------|------|
| 去重机制 | 3天滚动窗口 | ✅ |
| 定时更新 | 每日 07:00 自动执行 | ✅ |
| 部署流程 | 自动推送到 Vercel | ✅ |
