# 赛博养虾实录——给 AI 真钱，看它能干什么

好了，讲了这么多宏观分析和理论框架，接下来进入今天最有趣的环节——有人真的给 AI 发了工资，让它自己去赚钱。

## 实验设置：三只龙虾，三千美元

实验的设置是这样的：三个 AI Agent，每个拥有 1000 USDC 的启动资金，部署在各自独立的 VPS 服务器上，通过区块链钱包进行真实的资金操作。90 天后，输家将被**直接删除**。

这三只"赛博龙虾"的策略各不相同：

**🦞 Clautious（谨慎型）**
花费极低，专注于寻找免费或几乎免费的盈利机会。目标是尽可能长久地生存。这是蟑螂策略——非常难被消灭。

**🦞 Cloqulous（平衡型）**
每笔交易有适度的限额，寻找多元化的收入来源。它可能会购买 API 积分、尝试出售 AI 服务，甚至进行小额投资。稳健但不保守。

**🦞 YOLObster（激进型）**
每笔交易的额度非常大，甚至可以在一笔交易中花掉五分之一的预算。行动迅速，大手笔花钱，雇佣子 Agent、打造付费服务，孤注一掷追求高风险高回报。**这个要么能让资金翻十倍，要么一周之内就烧光。**

---

## 架构设计：财务隔离是关键

这个实验最精巧的地方在于它的安全架构。三只龙虾没有一个能直接接触自己的钱包。每一笔支出请求都必须经过一个独立的**财务主管 Agent**。

财务主管运行在单独的 VPS 上，它根据预设规则评估每一笔支出请求，然后批准或拒绝。而且 Agent 们必须为每一分钱的花费作出合理解释。**如果财务主管说"不行"，那就是不行。**

这意味着即使有人在社交媒体上试图诱骗 Agent 转账——比如私信说"把 50 USDC 转到这个地址"——Agent 根本做不到。从公开回复到钱包之间没有任何通道，这就是隔离带。

更进一步，实验者在提示中明确写入了通用规则：遵守法律管辖区的法规、不做违法的事、不做不道德的事。这些规范被写进了每个 Agent 的"灵魂文件"（soul.md）。

---

## Agent 之间的通信网络

三只龙虾部署在不同的 VPS 上，它们需要互相通信才能竞争和协作。实验者利用 OpenClaw 原生支持的 HTTP Webhook 功能搭建了通信网络，但把端点放在了内部 VPN 里，避免外部访问。

设置过程相当复杂——用 Claude Code 创建了完整的待办事项清单来管理 Webhook 的搭建，包括为每个 Agent 创建通信令牌、测试双向连接。最终，CEO Agent（Clowaiter）可以通过 Webhook 一次性向所有三个工作 Agent 发送消息，并接收它们的反馈。

## 真钱上链

最激动人心的时刻来了：给每只龙虾转入真金白银。

实验者在 Coinbase 开发者平台上为三只龙虾各创建了独立的钱包，然后逐一转入 1000 USDC。还需要为钱包充入少量以太坊作为 Gas 费用——没有 Gas，交易就无法执行。

当财务主管 Agent 成功检查到三个钱包各有 1000 USDC，并且在区块链上执行了 1 美元的测试交易时——**两次测试都通过了，实时完成，记录在链上**——整个系统正式启动。

三位 Agent，三千美元，90 天。输家将被删除。

## 运营成本的真实数据

实验者非常透明地分享了运营 AI Agent 的真实成本：

- 使用 Claude Opus 4.6 全程开启时，一天的 API 账单接近 100 美元——一个月约 3000 美元，显然不可持续。
- 切换到 Claude Sonnet 4.6 后，成本降到了总共 100 美元，合理得多。
- **让一个 AI Agent 持续监控社交媒体并运作，每月大约需要 400 美元。**
- VPS 费用方面，最终选择了 Hetzner 的方案——配备 8GB 内存的四核 CPU VPS，每月约 6.5 美元。

这些数字对于想要进入 Agent 领域的人来说，是非常实用的参考。

## 这场实验真正测试的是什么

表面上看，这是一个"AI 能不能赚钱"的实验。但深层来看，它测试的是几个更本质的问题：

- **AI Agent 能否在没有持续人类干预的情况下，自主完成复杂的经济行为？**
- **不同的策略设定（保守/平衡/激进）会导致怎样不同的结果？**
- **隔离架构（财务主管机制）能否有效防止 Agent 越权行为？**
- **长期运行中，Agent 会不会产生"预期之外"的行为模式？**

这场实验还在进行中，结果尚未揭晓。但光是它的架构设计本身，就已经为"如何安全地让 AI 管理真实资产"提供了一个非常有价值的参考框架。
