## 1️⃣ Shopee "+1" 直播文化适配 Skill

### 1.1 功能概述
监听 Shopee Live 直播间弹幕，识别台湾消费者习惯的 "+1" 下单意图（及变体如 "我要+1"、"+2"、"加一" 等），自动收集订单信息（商品SKU、数量、买家信息）。

### 1.2 技术可行性分析

#### ✅ 可行路径 A：官方 Open API（推荐）

**技术依据**：
- Shopee Open Platform 提供官方 API：`v2.livestream.get_latest_comment_list`
- **支持地区明确包含台湾(TW)**、印尼(ID)、泰国(TH)、菲律宾(PH)、马来西亚(MY)、新加坡(SG)、越南(VN)
- 可获取最近10秒内的评论数据，包含用户ID、用户名、评论内容、评论时间

**优势**：
- 稳定可靠，不受前端改版影响
- 合法合规，无封号风险
- 支持多地区统一接口

**限制**：
- 需要申请 Shopee Open Platform 开发者账号
- API 调用频率有限制（需确认具体配额）
- 需要商家授权获取店铺/直播间访问权限

#### ⚠️ 备选路径 B：浏览器自动化（Fallback）

**技术方案**：Playwright > Selenium

**原因**：
- Playwright 自动等待机制更强，减少 flaky 测试
- 原生支持网络拦截（可捕获 WebSocket 弹幕数据流）
- 多浏览器引擎支持（Chromium/Firefox/WebKit）

**实现要点**：
```python
# 核心流程示意
1. 启动浏览器 → 登录 Shopee 账号 → 进入直播间
2. 监听弹幕容器 DOM 变化 / 拦截 WebSocket 消息
3. NLP 识别 "+1" 变体 → 提取商品名、数量
4. 结构化存储订单信息
```

**挑战**：
- Shopee 有成熟的反爬虫机制（CAPTCHA、IP 限流、设备指纹）
- 需要模拟真实用户行为（随机滚动、鼠标移动、合理操作间隔）
- 页面结构变更需及时维护选择器

## 2️⃣ Facebook 直播弹幕监听 + 自动回复原型

### 2.1 功能概述
监听 Facebook Live 评论区，识别用户提问/下单意向，自动生成并发送繁体中文回复。

### 2.2 技术可行性分析

#### ✅ 官方 API 方案（强烈推荐）

**技术依据**：
- Meta Graph API 提供完善的 Live Video API
- 支持实时评论获取（Server-Sent Events / 轮询）
- 支持评论创建、编辑、删除、回复

**API 能力**：
```
GET /{live-video-id}/comments          # 获取评论
POST /{comment-id}/comments            # 回复评论
POST /{live-video-id}/comments         # 发送新评论
```

**实时监听方案**：
- **推荐**: Server-Sent Events (SSE) 端点 —— 更高效、避免限流
- **备选**: 反向时间序轮询（每1-2秒）

**权限要求**：
- `pages_read_engagement` —— 读取直播评论
- `pages_manage_posts` —— 发送评论回复
- 需要 Facebook App + Page Access Token

#### ⚠️ 浏览器自动化 Fallback

仅在 API 权限受限时使用，技术方案同 Shopee 部分。

## 4️⃣ 技术风险识别与应对

### 4.1 平台反爬虫/反自动化风险

| 平台 | 风控级别 | 检测手段 | 对抗方案 |
|:-----|:---------|:---------|:---------|
| **Shopee** | 🔴 极高 | 设备指纹、行为分析、CAPTCHA、IP限流 | 官方 API 优先；浏览器方案需：代理池、指纹伪装、行为模拟 |
| **Facebook** | 🟡 中高 | 账号行为异常检测、API 限流 | 官方 API 足够；避免浏览器方案 |

**综合应对策略**：
1. **官方 API 优先原则**：尽可能使用官方 API，避免对抗
2. **分层降级**：API 失败 → 浏览器自动化 → 人工兜底
3. **账号矩阵**：准备多组账号，轮换使用
4. **行为模拟**：真实鼠标轨迹、随机延迟、正常浏览行为
