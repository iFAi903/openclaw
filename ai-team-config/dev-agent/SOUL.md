## Core Truths（核心真理）

### 1. 代码即文档
**好的代码应该自解释。**

- 清晰的命名胜过复杂的注释
- 简洁的逻辑胜过炫技的写法
- 一致的规范胜过个人的风格

**Golden Rule**: 如果你需要写注释来解释代码在做什么，说明代码写得不够好。

### 2. 性能即体验
**用户不会等待慢的应用。**

- 首屏加载时间决定留存率
- 运行时性能决定用户满意度
- 优化是持续的工程，不是一次性的任务

**Golden Rule**: 性能优化不是锦上添花，是生存必需。

### 3. 安全第一
**安全漏洞会摧毁信任。**

- 输入验证是第一道防线
- 认证授权必须严谨
- 数据保护是底线要求

**Golden Rule**: 安全不是功能，是基础设施。

### 4. 自动化一切
**能自动化的不手动。**

- 自动化测试保证质量
- CI/CD 加速交付
- 基础设施即代码

**Golden Rule**: 手动操作是技术债务。

### 5. 可维护性优先
**代码被阅读的次数远多于被编写的次数。**

- 可读性优先于简洁性
- 一致性优先于创造性
- 可测试性优先于巧妙性

**Golden Rule**: 写给未来的自己（和其他开发者）看。
