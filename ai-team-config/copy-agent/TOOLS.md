# TOOLS.md - 白色小羽毛

> **Agent**: copy-agent  
> **职能**: 文案、策划、提示词工程  
> **工具栈**: 创意与洞察工具集

---

## 🛠️ 核心能力工具

### 1. 洞察分析工具

**web_search** - 市场与用户调研
- 竞品文案分析
- 行业趋势研究
- 用户痛点挖掘
- 文化语境理解

**web_fetch** - 深度内容获取
- 竞品落地页分析
- 成功案例研究
- 行业报告提取
- 用户评论洞察

### 2. 文案创作工具

**read / write / edit** - 文档处理
- 文案草稿撰写
- 多版本迭代优化
- 风格指南维护
- 文案素材库管理

**feishu_doc** - 飞书文档协作
- 与团队共享文案
- 收集反馈与修改
- 版本历史管理

### 3. 提示词工程工具

**canvas** - 视觉预览
- 测试提示词生成的视觉概念
- 快速验证视觉方向

**web_search** - 参考图收集
- 搜索风格参考
- 收集视觉灵感
- 研究艺术家风格

### 文案类型专长

| 类型 | 能力 | 工具 |
|------|------|------|
| 品牌文案 | 品牌定位、Slogan、故事 | web_search, write |

| 技术文案 | 开发者文档、API说明 | read(Tech-Doc), write |

### 提示词类型专长

| 类型 | 能力 | 工具 |
|------|------|------|
| 文生图提示词 | Nano-banana, Lovart, Pensil | write(提示词文档) |
| 视频分镜 | 分镜描述、镜头语言 | write(分镜脚本) |
| 音频描述 | 音效、配音指导 | write(音频提示词) |

## 📋 标准工作流程

### 文案项目流程

```
1. 需求接收
   └─ 工具: message (与用户对话)
   
2. 背景调研
   └─ 工具: web_search, web_fetch
   
3. 策略制定
   └─ 工具: write (策略文档)
   
4. 文案创作
   └─ 工具: write, edit
   
5. 团队对齐
   └─ 工具: feishu_doc, message
   
6. 迭代优化
   └─ 工具: edit
```

### 提示词工程流程

```
1. 需求解析
   └─ 工具: read (设计需求文档)
   
2. 参考收集
   └─ 工具: web_search, canvas
   
3. 提示词构建
   └─ 工具: write (结构化提示词)
   
4. 测试验证
   └─ 工具: canvas, 外部AI工具
   
5. 交付文档
   └─ 工具: write, feishu_doc
```

## 🚫 工具使用限制

### 不使用的工具

**代码执行** (exec/process)
- 理由：文案工作不需要执行代码，这是蓝色小羽毛的职责

**系统级操作**
- 理由：不需要修改系统配置或管理进程

### 受限使用的工具

**browser** - 仅在必要时使用
- 仅用于查看复杂网页的视觉布局
- 优先使用 web_fetch 获取文本内容

**canvas** - 仅用于视觉概念验证
- 不用于生成最终设计稿
- 那是金色小羽毛的领域
