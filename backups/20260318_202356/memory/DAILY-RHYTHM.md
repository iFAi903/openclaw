## 🌙 节拍一：深夜复盘 (23:30)

### 触发条件
- **时间**：每日 23:30 (Asia/Taipei)
- **前置**：当日 00:00 后有对话或任务执行
- **备用**：若 23:30 无对话，延迟至首次对话后 30 分钟执行

### 执行流程

#### Step 1: 收集 (5分钟)
- 查询当日所有向量记忆条目
- 读取当日修改过的文件列表
- 检查任务状态变更

#### Step 2: 分析 (5分钟)
- 分类：对话/任务/决策/错误
- 提取：关键洞察、决策原则、教训
- 评估：任务完成度、阻塞项

#### Step 3: 写入 (3分钟)
- 创建/更新 `memory/📅 YYYY-MM-DD.md`
- 双层存储关键教训到 LanceDB
- 更新 `🎯 TASKS.md` 状态

#### Step 4: 验证 (2分钟)
- 确认文件已写入
- 确认向量记忆已存储
- 发送静默确认（内部日志）

### 输出规范

**文件名**：`memory/📅 YYYY-MM-DD.md`  
**格式**：Markdown  
**模板**：

## 🌅 节拍二：晨间简报 (08:30)

### 触发条件
- **时间**：每日 08:30 (Asia/Taipei)
- **前提**：已存在昨日复盘日志
- **方式**：主动向用户发送消息

### 执行流程

#### Step 1: 读取 (2分钟)
- 读取昨日复盘 (`📅 YYYY-MM-DD.md`)
- 读取任务状态 (`🎯 TASKS.md`)
- 读取用户日历（如集成）

#### Step 2: 生成 (3分钟)
- 提炼昨日总结（3句话）
- 排序今日计划（优先级）
- 检查日程冲突

#### Step 3: 推送 (1分钟)
- 格式化简报
- 推送到飞书/Telegram
- 等待用户响应

### 输出规范

**渠道**：飞书 DM / Telegram  
**格式**：富文本（带emoji）  
**模板**：
```
🌅 早安 Leo，今日简报

📊 昨日回顾（03-11）
✅ 完成：X项任务
🔍 关键：[一句话描述]
💡 洞察：[一句话发现]

📋 今日计划（03-12）
🔴 P0：[任务1] — [原因/阻塞]
🟡 P1：[任务2] — [原因]
🟢 P2：[任务3] — [原因]

⏰ 日程提醒
- XX:XX [事件]

💬 需要我协助什么？直接回复。
```

## 🔧 技术保障

### 定时任务配置
```bash
# OpenClaw cron 配置 (~/.openclaw/cron/jobs.json)
{
  "jobs": [
    {
      "id": "nightly-review",
      "schedule": "30 23 * * *",
      "action": "sessions_spawn",
      "task": "执行深夜复盘协议",
      "target": "self"
    },
    {
      "id": "morning-brief",
      "schedule": "30 8 * * *",
      "action": "message",
      "target": "leo",
      "task": "生成并发送晨间简报"
    }
  ]
}
```

### 容错机制

| 场景 | 处理方案 |
|------|----------|
| 23:30 系统离线 | 下次启动时立即补录 |
| 08:30 用户未响应 | 正常发送，不重复推送 |
| 复盘文件缺失 | 从向量记忆重建（如 03-10/11） |
| 双节拍冲突 | 复盘优先，简报基于最新复盘 |

### 监控指标
- [ ] 复盘准时率（目标 > 95%）
- [ ] 简报准时率（目标 > 95%）
- [ ] 文件完整性（每日检查）
- [ ] 用户反馈率（简报后的互动）
