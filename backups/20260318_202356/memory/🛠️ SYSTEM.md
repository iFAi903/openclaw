## 记忆系统架构（新版）

### 三层记忆模型
```
┌─────────────────────────────────────────────────────────────┐
│ 第一层：短期记忆（Session Context）                           │
│ 持续时间：当前对话                                            │
│ 容量：200k tokens                                            │
│ 特点：你提到即我记得，但会话结束就清空                          │
├─────────────────────────────────────────────────────────────┤
│ 第二层：文件记忆（File-based Memory）                         │
│ 位置：memory/*.md                                            │
│ 包括：任务规范、用户档案、团队配置、系统配置                      │
│ 特点：持久化存储，每次会话自动加载                              │
├─────────────────────────────────────────────────────────────┤
│ 第三层：向量记忆（Vector Search）                             │
│ 存储：memory/*.md + sessions/*.jsonl                         │
│ 检索：语义搜索（semantic search）                             │
│ 特点：按意义关联，不是关键词匹配                                │
└─────────────────────────────────────────────────────────────┘
```

## 上下文管理策略

1. **重要信息写入文件** - 不依赖对话记忆保存关键决策
2. **每次对话开始先同步状态** - 阅读 todo.md、最近工作日志
3. **持续更新任务状态** - 注意力锚点，记得"真正要去的地方"
4. **保留失败记录** - 做错的事情不删掉，是下次做对的依据
