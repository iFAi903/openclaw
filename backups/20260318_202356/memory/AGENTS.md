```
用户输入
```

### 7. 端口冲突修复记录

**问题描述**：2026-02-25 发现所有 Agent 端口冲突，都指向 18789，导致无法同时启动

**根因**：
1. LaunchAgent plist 文件被错误配置为统一端口 18789
2. 存在 legacy 服务标签干扰（ai.openclaw.copy/product/design/dev）
3. ai-team-status.sh 依赖 lsof，但系统环境中不可用

**修复步骤**：
1. 停止并清理 legacy 服务标签
2. 修正每个 Agent 的 LaunchAgent plist 端口：
   - copy-agent: 18782
   - product-agent: 18785
   - design-agent: 18788
   - dev-agent: 18790
3. 重启所有 Agent 服务
4. 修复 ai-team-status.sh，添加 nc 回退检测
