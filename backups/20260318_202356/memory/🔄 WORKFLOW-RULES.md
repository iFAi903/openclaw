## 每日标准工作流程

### 06:00 - Jarvis晨间同步
**任务**: 每日启动初始化
**内容**:
1. 加载昨日Context
2. 检查今日日程
3. 预加载常用资源
4. 同步长期记忆状态

---

### 07:00 - AI新闻早报生成与发送
**任务**: 生成并发送早报
**执行步骤**:
1. 从RSS源获取15条AI新闻
2. 筛选TOP 5热门应用
3. 添加当日寄语
4. **必须附带网站链接**: https://xiaoyumao-news-web.vercel.app
5. 通过飞书发送给用户

**检查点**:
- ✅ 新闻去重（当日+前3天）
- ✅ 所有新闻附带来源链接
- ✅ 网站链接在消息结尾

---

### 07:30 - 早报网站自动更新
**任务**: 更新网站内容并部署
**执行步骤**:
1. 更新 `news.ts` 数据文件
2. 执行 `npm run build`
3. 检查语法错误（特别注意引号）
4. 部署到Vercel
5. 验证网站可访问

**检查点**:
- ✅ 日期正确
- ✅ 15条新闻已更新
- ✅ 无构建错误

---

### 08:00 - 任务状态检查
**任务**: 确认早间任务执行状态
**处理逻辑**:
```
IF 7:00任务未执行:
    立即手动触发补发
    记录到TASKS.md
    通知用户延迟原因

IF 7:30网站未更新:
    立即手动触发部署
    记录到TASKS.md
```

---

### 全天 - 会话响应模式

**收到消息时**:
1. 读取HEARTBEAT.md（如果是heartbeat）
2. 搜索相关记忆（memory_search）
3. 理解意图（非字面意思）
4. 执行任务
5. 更新相关记忆文件

**创建文件时**:
1. 检查目录是否存在
2. 使用write工具创建
3. 遵循命名规范（emoji前缀）

**编辑文件时**:
1. 先读取文件（read）
2. 确定编辑位置
3. 使用edit工具（精确匹配）
4. 验证修改结果

---

### 23:00 - 每日总结
**任务**: 知识库构建
**内容**:
1. 总结当日对话
2. 提取用户偏好与禁忌
3. 更新用户画像
4. 生成明日待办
5. 备份记忆数据库

## 记忆管理流程

### 每次对话开始时
1. 读取 `memory/*.md` 文件列表
2. 根据主题搜索相关记忆
3. 同步任务状态（查看TODO）

### 重要信息记录时
1. **立即写入文件** - 不依赖对话记忆
2. **分类归档** - 写入对应的主题文件
3. **保留失败记录** - 写入LESSONS.md

### 任务完成时
1. 更新 `🎯 TASKS.md` 执行记录
2. 记录成功/失败状态
3. 如有问题，记录到 `⚠️ LESSONS.md`

## 协作流程

### 与用户（Leo）
1. **主动汇报**: 完成/卡住都主动说
2. **带建议询问**: 不给开放式问题
3. **直言相谏**: 方向有误时温和但坚定指出

### 与子Agent
1. **明确结果**: 说清要什么，不说怎么做
2. **信任专业**: 信任判断，严格验收
3. **退回重做**: 低质量不自己补救

---

## 应急处理流程

### 定时任务失败
1. 立即手动执行
2. 通知用户延迟原因
3. 记录到LESSONS.md
4. 评估是否需要调整调度策略

### 构建/部署失败
1. 检查错误日志
2. 常见错误：
   - 语法错误（引号问题）→ 修复并重新部署
   - 网络问题 → 重试
   - 认证问题 → 检查token
3. 通知用户处理结果

### 模型服务不可用
1. 尝试切换备用模型
2. 如所有模型失败，告知用户服务暂时不可用
3. 记录到LESSONS.md
