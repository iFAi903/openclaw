## 📋 核心规则

### 1. 标准项目流程

```
用户输入
    ↓
[小羽毛-main] 接收需求，分析任务类型
    ↓
    ├─→ 创意/文案需求 ──→ [白色小羽毛]
    ├─→ 产品/规划需求 ──→ [紫色小羽毛]
    ├─→ 设计/视觉需求 ──→ [金色小羽毛]
    └─→ 开发/技术需求 ──→ [蓝色小羽毛]
    ↓
各 Agent 完成专业任务
    ↓
[小羽毛-main] 整合输出，质量把控
    ↓
交付用户
```

**关键原则**：
- **单点接触**：用户只与 main 交互
- **专业分工**：每个 Agent 专注核心领域
- **质量把关**：main 负责最终输出质量

---

### 2. 交接标准

| 交接方向 | 交付物 | 验收标准 | 接口方式 |
|----------|--------|----------|----------|
| **白色 → 紫色** | 创意概念 + 策划案 | 需求清晰、目标明确 | `sessions_send` |
| **紫色 → 金色** | PRD文档 | 需求完整、逻辑清晰 | `sessions_send` |
| **金色 → 蓝色** | 设计规范 + 原型 | 规格明确、可实现 | `sessions_send` |
| **蓝色 → 用户** | 可部署应用 | 功能完整、质量达标 | main 整合后交付 |

**交接规范**：
1. 明确发送者和接收者
2. 简洁明了，直奔主题
3. 提供必要的上下文（`context` 字段）
4. 关键节点需确认接收

---

### 3. Agent间通信协议

**输入接口**（每个 Agent 必须支持）：
- `message` - 接收任务描述
- `file` - 接收相关文档
- `context` - 接收上下文信息

**输出接口**（每个 Agent 必须提供）：
- `deliverable` - 交付物（文档/代码/设计等）
- `status` - 任务状态更新
- `handoff` - 交接给下一个 Agent 的信息

**通信工具**：
```python
sessions_send(sessionKey, message, context)
```

### 5. 定时任务流程

**每日 07:00 - AI新闻早报**：
```
[cron触发]
    ↓
[isolated session: main]
    ↓
[任务1] 获取15条AI新闻
[任务2] 获取TOP 5热门应用
[任务3] 生成今日寄语
    ↓
[发送] 飞书消息给用户
    ↓
[状态报告] 返回执行结果
```

**每日 07:30 - 网站自动更新**：
```
[cron触发]
    ↓
[isolated session: main]
    ↓
[更新] news.ts 数据文件
[构建] npm run build
[部署] vercel --prod
    ↓
[状态报告] 返回部署结果
```

**关键规则**：
- 定时任务使用 `isolated` sessionTarget
- 每个任务独立执行，失败不影响其他任务
- 执行结果通过 `delivery.announce` 发送给用户

### 6. 异常处理流程

**Agent故障**：
1. 自动重启该 Agent 的 Gateway
2. 通知小羽毛-main
3. 评估是否需要其他 Agent 接管

**任务阻塞**：
1. 及时上报阻塞原因
2. 小羽毛-main 协调资源
3. 必要时调整方案

**质量问题**：
1. 验收不通过时退回修改
2. 记录质量问题和改进措施
3. 更新工作规范

---

### 7. 质量检查清单

**交付前必须检查**：
- [ ] 是否符合任务要求
- [ ] 是否经过相关专业 Agent 处理
- [ ] 输出格式是否规范
- [ ] 是否有明确的下一步行动
- [ ] 用户是否可以直接使用

**多 Agent 协作检查**：
- [ ] 交接信息是否完整
- [ ] 上下文是否传递正确
- [ ] 各专业输出是否一致
- [ ] 整合后的输出是否连贯

### 8. 启动/停止流程

**启动AI天团**：
```bash
~/ai-team-start.sh
```
执行顺序：main → copy-agent → product-agent → design-agent → dev-agent

**停止AI天团**：
```bash
~/ai-team-stop.sh
```

**查看状态**：
```bash
~/ai-team-status.sh
```

---

### 9. 工作流优化原则

1. **并行化**：可并行任务同时分派
2. **最小交接**：减少不必要的 Agent 间传递
3. **提前确认**：关键决策点需用户确认
4. **记录沉淀**：常见问题形成规则文档
5. **持续改进**：根据反馈优化流程
